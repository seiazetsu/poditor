# Poditor 仕様書

## 1. 仕様書の目的

本書は Poditor の画面、機能、データ構造、基本的な振る舞いを明確化するための仕様書である。
実装時には `AGENTS.md` と `docs/requirements.md` をあわせて参照すること。

---

## 2. 技術仕様

- フロントエンド: Next.js App Router
- 言語: TypeScript
- UI: Chakra UI
- 認証: Firebase Authentication
- データベース: Firestore
- デプロイ: Vercel

---

## 4. 画面仕様

## 4.1 ログイン画面

### 目的

既存ユーザーがサインインするための画面

### 表示項目

- メールアドレス入力欄
- パスワード入力欄
- ログインボタン
- 新規登録画面への導線
- エラーメッセージ表示領域

### 動作

- 正常な認証情報の場合、ログイン後画面へ遷移する
- 認証失敗時、エラーメッセージを表示する

---

## 4.2 ユーザー登録画面

### 目的

新規ユーザーがアカウントを作成するための画面

### 表示項目

- メールアドレス入力欄
- パスワード入力欄
- 登録ボタン
- ログイン画面への導線
- エラーメッセージ表示領域

### 動作

- 正常登録後はログイン済み状態とし、プロジェクト一覧画面へ遷移させる
- 入力不備や登録失敗時はエラーメッセージを表示する

---

## 4.3 プロジェクト一覧画面

### 目的

ユーザーが参加しているプロジェクトを一覧表示し、プロジェクト選択・作成へつなげる

### 表示項目

- 画面タイトル
- ログイン中ユーザー情報の簡易表示
- ログアウトボタン
- 新規プロジェクト作成ボタン
- プロジェクト一覧

### プロジェクト一覧の各項目

- プロジェクト名
- 更新日時
- プロジェクト詳細画面への導線

### 動作

- ログイン中ユーザーが参加しているプロジェクトのみ表示する
- 一覧は更新日時の降順を基本とする

---

## 4.4 プロジェクト詳細画面

### 目的

選択したプロジェクトの概要と参加メンバー、およびそのプロジェクトに属する台本一覧を表示する

### 表示項目

- プロジェクト名
- 設定メニュー
- 台本一覧
- プロジェクト一覧へ戻る導線

### 設定メニューの内容

- 参加メンバー一覧
- 参加者追加フォーム

### 台本一覧の各項目

- タイトル
- ステータス
- 更新日時
- 台本基本設定ページへの導線
- 台本複製ボタン
- 台本削除ボタン

### 動作

- 開いたプロジェクトに属する台本のみ表示する
- 参加中ユーザーはそのプロジェクト内の台本を閲覧・編集できる
- owner はプロジェクト本体と参加メンバーを管理できる
- member はプロジェクト内台本の閲覧・編集を行える
- プロジェクト削除導線は owner にのみ表示する
- 参加メンバー関連 UI は設定メニュー内に格納する
- 台本複製時は、台本本体に加えて参考文献、話者、items を新しい台本として複製する
- 台本一覧は Google Drive のような密な1行リストを基本とし、1行内にタイトル、更新日時、操作アイコンを表示する
- 新規台本作成は、台本一覧見出し行の右端にアイコンボタンで表示する
- 台本一覧では `全て / 下書き / 完成 / 収録済` のステータス絞り込みを行える
- 台本一覧のステータスバッジはクリックで `下書き → 完成 → 収録済 → 下書き` の順に循環更新する
- 台本タイトルをシングルクリックすると会話作成ページへ遷移する
- 台本タイトルはダブルクリックでインライン編集できる

---

## 4.5 台本基本設定ページ

### 目的

台本の土台を作成するページ。
ここではタイトルと話者のみを扱い、会話本文はまだ作成しない。

### 表示項目

- タイトル入力欄
- ステータス選択欄
- 話者一覧
- 話者追加フォーム
- 参考文献一覧
- 会話作成ページへ進むアイコンボタン
- プロジェクト詳細画面へ戻るアイコン導線

### 動作

- タイトルを編集できる
- タイトル入力欄の右側に保存ボタンを横並びで配置する
- ステータスを `下書き` `完成` `収録済` から選択して保存できる
- 閲覧専用の共有プレビューリンクを更新してコピーできる
- 既存の共有プレビューリンクをコピーできる
- 共有プレビューを無効化できる
- 話者を追加、編集できる
- 話者追加・編集 UI は 1 行横並びを基本とする
- 話者を削除できる
- ただし、その話者を参照する dialogue 行が存在する場合は削除操作時にエラーを表示し、削除できない
- 少なくとも1人以上の話者がいることが望ましい
- 会話本文はこのページでは扱わない
- タイトル保存、会話作成への遷移、戻る、話者追加・保存・削除はアイコンボタンで行う
- 「会話作成ページへ進む」で次ページへ遷移する
- 第1段階の完了画面として扱う
- レイアウトは広めの 2 カラムとし、左にタイトルと話者、右に参考文献を配置する
- スマートフォン幅では、タイトル保存、ステータス保存、共有プレビュー操作を縦積み中心に切り替え、横にはみ出さないこと
- 参考文献の一括コピーはアイコン操作で行う

---

## 4.6 会話作成ページ

### 目的

第2段階として、セリフを追加、編集、並び替えしながら完成イメージを見て台本を作るページ

### レイアウト

左右2カラム構成とする。

#### 左カラム

- 話者選択とセリフ投稿を行う入力フォームエリア
- セクション追加フォームを含む
- メディア投稿フォームを含む
- 挿入位置指定 UI を含む
- 基本設定ページへ戻る導線
- プロジェクト詳細へ戻る導線
- 共有プレビューを開く導線
- 会話ビュー印刷ボタン
- テキストエクスポートボタン

#### 右カラム

- セクション、セリフ、メディアを同一順序で表示する台本表示エリア
- 台本タイトルの下に概算時間を小さく表示する
- 複数選択モードの切り替えと一括削除を行える
- 右上にメモビュー開閉アイコンを表示する
- アイコン押下時は画面右側にメモビューを開く
- メモビューは参考情報を自由入力できるテキストエリアとする
- 右下に参考文献登録アイコンを表示する
- アイコン押下時は下から参考文献登録フィールドを開く
- 参考文献登録フィールドでは文字列と URL を登録できる
- スマートフォンでは、メモビューと参考文献登録フィールドを右スワイプで閉じられる

左右のカラムはそれぞれ独立して縦スクロールできること。
メモビューを開いた場合も、会話ビューとメモビューは独立して縦スクロールできること。
印刷時は左カラム、メモビュー、挿入 UI、削除 UI を非表示にし、会話ビューのみを出力すること。
スマートフォン表示では、会話ビューを左フォームより先に表示し、メモビューは全画面に近いオーバーレイで開けること。

---

## 4.7 共有プレビュー画面

### 目的

台本を書かない閲覧者に、ログイン不要の閲覧専用リンクを共有するための画面

### 表示項目

- 台本タイトル
- 概算時間
- セクション
- セリフ
- メモ
- 画像または URL メディア
- プレビュー更新日時
- 右上固定の URL コピーボタン

### 動作

- `/preview/{token}` 形式の URL でアクセスできる
- ログイン不要で閲覧できる
- 編集、削除、並び替え、挿入などの編集 UI は表示しない
- token が無効または停止済みの場合はエラー表示にする

---

## 5. 会話作成ページ詳細仕様

## 5.1 左カラム（入力フォームエリア）

### 表示項目

- 話者選択
- 入力モード切り替え
- セクション入力欄
- セリフ入力欄
- セリフ追加ボタン
- メディア種別選択
- メディア名入力欄
- メディア URL 入力欄
- メディア追加ボタン

### 動作

- 話者を選択できる
- 入力モードを `セリフ入力` と `セクション入力` で切り替えできる
- 初期表示は `セリフ入力` とする
- 入力モード切り替えは話者選択より上に配置する
- `セクション入力` 選択時は話者選択 UI を表示しない
- 話者選択 UI の末尾には灰色の `メモ` 選択肢を表示する
- セクション名を入力して見出し行を追加できる
- セクション追加完了後は入力モードを `セリフ入力` に戻す
- セリフ本文を入力できる
- セリフ入力欄では Shift+Enter で投稿できる
- デフォルトでは末尾に追加する
- 任意の位置を選択した場合、その位置に挿入する
- 左カラム内だけで操作が完結すること
- 画像または URL メディアを追加できる
- メディア追加時もセリフと同じ挿入位置ルールを使う
- 画像はドラッグ&ドロップまたはファイル選択で添付できる
- 画像は添付時に Storage へアップロードし、`セリフを投稿` または Shift+Enter のタイミングで本文項目として追加する

### 挿入位置仕様

初期状態では「末尾に追加」とする。
任意位置指定時は、右カラム上の挿入対象位置を選べるようにする。
左カラムには挿入位置専用の常時表示 UI を置かず、右カラム側の挿入アイコンで指定する。

### エクスポート仕様

- 会話作成ページから台本をテキスト形式でエクスポートできる
- エクスポート内容は画面上の並び順に従う
- セクションは `------------------------` の罫線で上下を囲んで出力する
- メモは `*************************` の罫線で上下を囲み、1 行目に `<メモ>` を出力する
- 各セリフブロックの間には 1 行空行を入れる
- 添付メディアがある場合は同じブロック内に URL を改行して出力する

---

## 5.2 右カラム（台本表示エリア）

### 表示内容

- セリフ行は左側の縦線で話者色を示す
- セクション行は通常のセリフより大きい文字と太めの左線を持つ見出しスタイルで表示する
- メディア行の左側縦線も、紐づく話者と同じ色を使う
- 各行は全て左寄せで表示する
- 縦線の右側に話者名と本文を表示する
- 話者名クリックで登録順に次の話者へ切り替える
- 3人以上でも全員左寄せのまま表示する
- メディア行はセリフ行と同じ一覧の中に表示する
- 画像はインラインプレビュー表示する
- 動画 URL はサムネイルまたはアイキャッチを表示する
- 画像タイトルのデフォルト文言は表示しない
- 概算時間は dialogue 本文の総文字数を空白除去後に合計し、`300文字/分` で算出する

### 行の表示構成

各セクション行は少なくとも以下を持つ:

- タイトル
- 並び順情報（内部的）
- ドラッグ対象としての識別子

各セリフ行は少なくとも以下を持つ:

- 話者名
- 本文
- 並び順情報（内部的）
- ドラッグ対象としての識別子

各メディア行は少なくとも以下を持つ:

- mediaType
- label
- url
- プレビューまたはアイキャッチ
- ドラッグ対象としての識別子

## 5.3 参考文献登録

### 登録 UI

- 右下の参考文献アイコンで開閉する
- 開いた状態では、文字列入力欄と URL 入力欄を表示する
- 再度アイコンを押すと閉じる
- URL は登録時に妥当な形式かを確認する
- 登録済み参考文献ごとに編集と削除をアイコン操作で行える

### 保存仕様

- 参考文献は台本ドキュメントに紐づく配列として保存する
- 1件ごとに `id`, `text`, `url` を持つ
- 登録順を維持して表示する

### 表示仕様

- 台本基本設定ページに参考文献一覧を表示する
- URL を持つ項目はリンク表示にする
- 台本基本設定ページでは編集 UI を持たない
- 台本基本設定ページでは参考文献全体を一括コピーできる
- 台本基本設定ページでは個別カードを使わず、枠のないリッチテキスト風の連続表示にする

### 動作

- セクション行、セリフ行、メディア行を一覧表示する
- セクション行、セリフ行、メディア行をまとめてドラッグアンドドロップで並び替えできる
- 選択モード中は項目クリックで選択でき、Shift+クリックで連続範囲選択できる
- 選択した複数項目を一括削除できる
- 挿入位置選択中は、どこに入るか分かる表示を行う
- 投稿後は右カラムの表示に反映される
- セリフ、メモ、またはセクションの投稿後は、追加されたブロックの末尾が見える位置まで右カラムを自動スクロールする
- セクション行はダブルクリックでインライン編集できる
- セリフ行はダブルクリックでインライン編集できる
- セリフ編集用の textarea は本文量に応じて自動で高さを伸縮させる
- 編集中は Shift+Enter で保存、Esc でキャンセルできる
- セクション行、セリフ行、メディア行は削除ボタンで削除できる
- セリフ行と添付メディア行はペアとして扱い、片方を削除した場合は相方も同時に削除する
- 右カラムを上方向へスクロールしている間だけ、最下まで戻る固定ボタンを右ビュー下中央に表示する

---

## 5.3 スクロール仕様

- 左カラムは独立して縦スクロールできる
- 右カラムも独立して縦スクロールできる
- 一方をスクロールしても他方は同期しない

---

## 5.4 セリフ追加仕様

### データ

セクション行は少なくとも以下を持つ:

- id
- type = `section`
- title
- order
- createdAt
- updatedAt

セリフ行は少なくとも以下を持つ:

- id
- speakerId
- content
- order
- createdAt
- updatedAt

### 追加処理

- セクション名が入力された状態でセクション追加可能
- 挿入位置未指定時は末尾追加
- 挿入位置指定時はその位置に挿入
- 必要に応じて後続行の order を再計算する
- 追加結果は右カラムの見出し表示に反映する
- 話者と本文が入力された状態で追加可能
- 挿入位置未指定時は末尾追加
- 挿入位置指定時はその位置に挿入
- 必要に応じて後続行の order を再計算する
- 追加結果は右カラムの縦線表示に反映する

### メディア行データ

メディア行は少なくとも以下を持つ:

- id
- type
- mediaType
- label
- url
- note
- order
- createdAt
- updatedAt

### メディア追加処理

- URL が入力された状態で追加可能
- 画像と URL メディアを区別して追加できる
- 画像アップロード前に、最大辺 500px・画質 50% 相当へ最適化する
- 挿入位置未指定時は末尾追加
- 挿入位置指定時はその位置に挿入
- 必要に応じて後続項目の order を再計算する
- 追加結果は右カラムの一覧表示に反映する

---

## 5.5 並び替え仕様

- 右カラム上のセクション行・セリフ行・メディア行はドラッグアンドドロップで並び替え可能
- 並び替え後は order を更新する
- 並び替え結果は Firestore に保存する
- 並び替え後の表示順はそのまま維持される

---

## 5.6 話者色表示仕様

- 話者ごとに設定した color をセリフ行左側の縦線に反映する
- 本文領域自体は可読性優先の背景で表示する
- 3人以上でもレイアウトが崩れないこと

---

## 6. 認証・アクセス制御仕様

### 6.1 ログイン状態判定

- Firebase Authentication の認証状態を用いる

### 6.2 保護対象

以下の画面はログインユーザーのみアクセス可能とする

- プロジェクト一覧画面
- プロジェクト詳細画面
- 台本基本設定ページ
- 会話作成ページ

### 6.3 未ログイン時の挙動

- 保護対象ページにアクセスした場合はログイン画面へ誘導する

### 6.4 プロジェクト参加者ベースのアクセス制御

- ユーザーは参加しているプロジェクトのデータにのみアクセスできる
- 参加していないプロジェクトのデータにはアクセスできない
- projects は参加者のみ read 可能とする
- project 本体の write と members の管理は owner のみ可能とする
- 参加中プロジェクト内の台本、話者、セリフは owner と member のどちらも閲覧・編集できる
- 初期段階では scripts 配下の細かな権限制御は行わない

---

## 7. データモデル仕様

## 7.1 projects コレクション

### ドキュメント項目

- id
- name
- ownerUid
- createdAt
- updatedAt

### 説明

- プロジェクト本体を保持する
- ownerUid は Firebase Authentication の uid
- 初期段階では owner はプロジェクト作成者を指す

---

## 7.2 projects/{projectId}/members サブコレクション

### ドキュメント項目

- id
- uid
- email
- role
- createdAt
- updatedAt

### 説明

- プロジェクト参加者を保持する
- role は初期段階では `owner` または `member` のいずれかとする
- uid は Firebase Authentication の uid と一致する

---

## 7.3 projects/{projectId}/scripts サブコレクション

### ドキュメント項目

- id
- title
- createdAt
- updatedAt

### 説明

- 台本は必ずどこか1つのプロジェクトに属する
- 台本の単独所有は行わない
- 台本へのアクセス権は親プロジェクトの参加者判定に従う

---

## 7.4 projects/{projectId}/scripts/{scriptId}/speakers サブコレクション

### ドキュメント項目

- id
- name
- color
- createdAt
- updatedAt

### 説明

- 台本ごとの話者を保持する
- color は表示用に使う

---

## 7.5 projects/{projectId}/scripts/{scriptId}/items サブコレクション

### ドキュメント項目

- id
- type
- order
- createdAt
- updatedAt

`type=dialogue` の場合:

- speakerId
- content

`type=section` の場合:

- title

`type=media` の場合:

- mediaType
- label
- url
- note

### 説明

- 会話作成ページではセクション行、セリフ行、メディア行を同一コレクションで管理する
- `type` は `section`、`dialogue`、`media` を使う
- `order` はセリフとメディアを通した並び順を表す
- セット投稿されたセリフ行とメディア行は、同じペア識別子を持てること

## 8. Firestore セキュリティ方針

### 基本方針

- 認証済みユーザーのみアクセス可能
- 参加中プロジェクトのデータのみ読み書き可能

### 実装方針

- projects/{projectId}/members に自分の uid が存在するかで参加者判定を行う
- projects 本体は参加者のみ read、owner のみ write とする
- members は参加者のみ read、owner のみ create/update/delete とする
- scripts、speakers、items は親プロジェクトの参加者判定に紐づいて read/write を許可する

## 9. バリデーション仕様

## 9.1 認証

- メールアドレスは必須
- パスワードは必須

## 9.2 台本

- タイトルは必須
- 長すぎる入力は制限してよい

## 9.3 話者

- 名前は必須
- 色は空でないこと

## 9.4 セリフ行

- 話者は必須
- 本文は空文字を許可しない

- 初期段階では、まず確実に使えることを優先する
