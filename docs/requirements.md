# Poditor 要件定義書

## 1. プロジェクト概要

### 1.1 システム名

Poditor

### 1.2 システムの目的

Poditor は、ポッドキャスト収録用の台本を作成・管理・閲覧するためのウェブアプリケーションである。

単なる文章エディタではなく、収録時に手元資料として使いやすいことを重視し、複数話者のセリフ管理、視認性の高い表示、本文の柔軟な挿入と並び替えを可能にする。

また、台本を単独で管理するのではなく、複数人で参加できる「プロジェクト」の中で管理できる構造とする。

### 1.3 想定ユーザー

- ポッドキャスト配信者
- 複数人で収録を行うユーザー
- 台本を事前に準備して収録に臨みたいユーザー
- 収録時に読みやすい画面を見ながら進行したいユーザー

---

## 2. 開発方針

### 2.1 基本方針

- 小さな単位で段階的に開発する
- 生成AIが継続的に要件と仕様を参照しながら実装する
- 人間が成果物を確認し、修正・改善指示を出す前提で進める
- 可読性、保守性、レビューしやすさを重視する
- 実際の台本制作体験を重視する

### 2.2 技術方針

- Next.js を使用する
- TypeScript を使用する
- App Router を使用する
- UI には Chakra UI を使用する
- Firebase をバックエンドとして利用する
- 認証には Firebase Authentication を利用する
- データ保存には Firestore を利用する
- デプロイ先は Vercel を想定する

## 3. 対象範囲

### 3.1 本プロジェクトで実装する範囲

- ユーザー登録機能
- ログイン機能
- ログアウト機能
- プロジェクトの作成、一覧表示、詳細表示
- プロジェクト参加者の管理
- プロジェクト内での台本の作成、一覧表示、編集、削除
- プロジェクト自体の削除
- 台本タイトルの管理
- 台本ごとの話者情報の管理
- 話者ごとのセリフ入力
- 話者ごとの色分け表示
- セリフの任意位置挿入
- セリフのドラッグアンドドロップ並び替え
- 画像および URL メディアの挿入
- 収録時に見やすい縦線表示型の台本画面

### 3.2 現時点で対象外とする範囲

- 音声収録機能
- 音声編集機能
- 自動文字起こし
- AIによる台本自動生成
- リアルタイム共同編集
- 高度な権限管理
- 課金機能
- SNSログイン
- オフライン対応
- モバイルアプリ化

## 4. 機能要件

## 4.1 認証機能

### 4.1.1 ユーザー登録

- ユーザーはメールアドレスとパスワードでアカウントを作成できること
- 不正な形式の入力に対してエラーメッセージを表示できること

### 4.1.2 ログイン

- ユーザーはメールアドレスとパスワードでログインできること
- 認証失敗時に適切なエラーメッセージを表示できること

### 4.1.3 ログアウト

- ログイン中のユーザーはログアウトできること

### 4.1.4 アクセス制御

- 未ログインユーザーはプロジェクト管理画面および台本管理画面にアクセスできないこと
- ログイン状態に応じて画面遷移を制御すること

---

## 4.2 プロジェクト管理機能

### 4.2.1 プロジェクト一覧

- ログインユーザーは、自分が参加しているプロジェクト一覧を確認できること
- 各プロジェクトについて最低限以下を表示できること
  - プロジェクト名
  - 更新日時

### 4.2.2 プロジェクト詳細

- ユーザーはプロジェクトを開き、その中の台本一覧を確認できること
- プロジェクト詳細画面では最低限以下を表示できること
  - プロジェクト名
  - そのプロジェクトに属する台本一覧
- 参加メンバー一覧と参加者追加 UI は設定メニュー内に格納できること

### 4.2.3 プロジェクト参加者

- 1つのプロジェクトには複数のユーザーが参加できること
- 初期段階では参加者の役割は以下の2種類とすること
  - owner
  - member
- owner はプロジェクト作成者とすること
- member は owner 以外の参加者とすること
- 初期段階では owner と member の細かな権限制御は分けず、どちらもそのプロジェクト内の台本を閲覧・編集できること

### 4.2.4 プロジェクト削除

- owner はプロジェクトを削除できること
- 削除時は確認操作を入れること
- 削除時は、そのプロジェクト配下の台本、話者、本文項目、参加者関連データもあわせて削除されること

---

## 4.3 台本管理機能

### 4.3.1 台本一覧

- ユーザーは参加中プロジェクトの中で台本一覧を確認できること
- 台本は必ずどこか1つのプロジェクトに属すること
- 台本一覧から既存台本を複製できること
- 台本一覧は1行に主要情報と操作を収め、縦方向にできるだけ多く表示できること
- プロジェクト詳細の台本一覧では、台本タイトルをダブルクリックして編集できること
- プロジェクト詳細の台本一覧では、台本タイトルをシングルクリックすると会話ページへ遷移できること
- プロジェクト詳細の台本一覧では、下書き・完成・収録済・全てで絞り込みできること
- プロジェクト詳細の台本一覧では、ステータスのバッジをクリックすると `下書き → 完成 → 収録済 → 下書き` の順で切り替えられること
- 各台本について最低限以下を表示できること
  - タイトル
  - ステータス
  - 更新日時

### 4.3.2 台本作成フロー

台本作成は2段階で行うこと。

#### 第1段階

- 台本基本設定ページで作成する
- タイトルを入力する
- 話者を登録する
- この段階では本文は作成しない
- 登録完了後に会話作成ページへ進めること

#### 第2段階

- 別ページの会話作成画面へ進む
- 会話作成画面で実際のセリフを追加、編集、並び替えする
- 第1段階で登録したタイトルと話者を前提に会話を作成する

### 4.3.3 台本編集

- タイトルを変更できること
- 台本のステータスを `下書き` `完成` `収録済` から設定できること
- 話者を追加、編集できること
- 台本基本設定ページの操作ボタンは、できるだけ文字列ではなくアイコンで簡潔に表示すること
- 台本を複製した場合、参考文献・話者・本文項目も含めて複製されること
- セリフを追加、編集、削除できること
- セクション見出しを追加、並び替え、削除できること
- セクション見出しをダブルクリックで編集できること
- セリフの順序を変更できること
- 会話作成画面では複数の項目を選択して一括削除できること

### 4.3.4 台本削除

- 参加者は参加中プロジェクト内の台本を削除できること

---

## 4.4 話者管理機能

### 4.4.1 話者設定

- 台本ごとに複数の話者を設定できること
- 話者には最低限以下の情報を持たせること
  - 名前
  - 表示色
- 基本設定ページでは話者管理フォームを 1 行横並び中心で表示できること
- 話者を削除できること
- ただし、会話に使用済みの話者は削除操作時にエラーを表示し、削除できないこと

### 4.4.2 話者数

- 話者数は台本ごとに異なってよい
- 将来的な増減に対応しやすい構造であること

---

## 4.5 会話作成機能

### 4.5.1 画面構成

会話作成画面は左右2分割とする。

#### 左側

- 話者選択フォーム
- セリフ入力フォーム
- セクション追加フォーム
- メディア投稿フォーム
- 挿入位置指定機能
- 会話ビュー印刷ボタン
- テキストエクスポートボタン
- 左カラム単体で縦スクロールできること

#### 右側

- 台本表示エリア
- セリフとメディアを一覧表示する
- セクション見出しも同じ order 系列で一覧表示する
- 台本タイトル下に概算収録時間を小さく表示できること
- 右カラム単体で縦スクロールできること
- 印刷時は会話ビューのみを出力できること
- 右カラムでは選択モードで複数の項目を選択できること
- 右上のアイコン操作でメモビューを開閉できること
- メモビューは画面右側に開き、会話ビューとは独立して縦スクロールできること
- メモビューには自由入力できるテキストエリアを持つこと
- 右下のアイコン操作で参考文献登録 UI を開閉できること
- 参考文献登録 UI は下から開く簡易登録フィールドであること
- 参考文献として文字列と URL を登録できること
- 参考文献の編集・削除はアイコン操作で行えること

#### 共通

- 左右は独立して縦スクロールできること

### 4.5.2 セリフ投稿

- 話者を選択してセリフを追加できること
- セクション名を入力してセクション見出しを追加できること
- 入力モードはセリフ入力とセクション入力を切り替えできること
- デフォルトはセリフ入力であること
- 入力モード切り替えは話者選択より上に表示すること
- セクション入力中は話者選択 UI を表示しないこと
- セクション追加後はセリフ入力に自動で戻ること
- 話者選択 UI の末尾に `メモ` を追加し、選択時は灰色のセリフとして投稿できること
- 投稿位置はデフォルトで末尾とする
- 任意の位置を選択してその位置に挿入できること
- セリフ入力欄では Shift+Enter で投稿できること
- セリフまたはセクションの投稿後は、会話ビューが末尾までスクロールすること
- 会話ビューでは選択モード中に項目クリックで複数選択できること
- Shift+クリックで連続範囲選択できること
- テキストエクスポート時、セクションは上下に罫線を付けて出力できること
- テキストエクスポート時、メモは `*************************` で上下を囲み `<メモ>` 見出し付きで出力できること

### 4.5.3 台本表示

- セリフ行は左側の縦線で話者色を表示すること
- セクション行は見出しとして視認しやすく表示できること
- 縦線は各行の高さに応じて表示されること
- 縦線の右側に話者名と本文を表示すること
- 話者名クリックで登録順に次の話者へ切り替えられること
- 複数話者でも全員左寄せで表示すること
- 3人以上でも表示が破綻しないこと
- 会話の並び順が視覚的に分かりやすいこと
- 画像メディアはプレビュー表示できること
- 動画 URL はアイキャッチまたはサムネイルを表示できること
- メディア行の左側縦線も、その直前に投稿したセリフ話者と同じ色で表示できること
- 画像アップロード時は最大辺 500px、画質 50% 程度に最適化してから保存すること
- 概算収録時間は dialogue 本文の総文字数を基に `300文字/分` で算出すること
- テキストエクスポート時は各セリフブロックの間に 1 行空行を入れること

### 4.5.4 並び替え

- セクション行、セリフ行、メディア行はドラッグアンドドロップで並び替えできること
- 並び替え結果は会話作成ページ上ですぐ反映されること

### 4.5.5 メディア投稿

- 画像と URL を会話本文の一部として投稿できること
- 画像はドラッグ&ドロップでアップロードできること
- メディアは `items` に保存し、セリフと同じ order で管理すること
- セクションも `items` に保存し、セリフやメディアと同じ order で管理すること
- 投稿済みメディアはセリフと同じ一覧の中で表示、並び替え、削除できること

### 4.5.6 参考文献登録

- 会話作成画面から参考文献を登録できること
- 会話作成画面から参考文献を修正、削除できること
- 参考文献は文字列のみ、URLのみ、または両方で登録できること
- URL を登録した場合はリンクとして扱えること
- 登録済み参考文献は台本基本設定ページに一覧表示できること
- 台本基本設定ページでは表示のみとし、その場では編集しないこと
- 台本基本設定ページでは参考文献全体を一括でコピーでき、その操作はアイコンで行えること
- 台本基本設定ページの参考文献表示は、個別カードではなく枠のない読み物風表示とすること

### 4.3.5 台本基本設定画面レイアウト

- 台本基本設定画面は広めの 2 カラムで表示できること
- 左カラムにタイトル設定と話者設定を配置すること
- 右カラムに参考文献リストを配置すること
- メディア単独ではなく、セリフ投稿時に一緒に追加できること
- セリフと添付メディアはセットとして扱い、どちらかを削除した場合は相方も同時に削除されること

### 4.5.6 編集操作

- セリフ行はダブルクリックでインライン編集できること
- セリフ編集時のフォームは本文量に応じて縦方向に伸縮し、元のセリフ量に見合った高さで表示されること
- 編集中は Shift+Enter で保存できること
- 編集中は Esc でキャンセルできること

## 5. 非機能要件

### 5.1 可読性

- 収録時に視認しやすいUIであること
- 長めの台本でも扱えること
- 10分から20分程度の収録台本を主な想定とするが、より長い台本にも対応できること

### 5.2 操作性

- 台本制作時に入力と完成イメージを同時に確認しやすいこと
- どこに追加されるか分かりやすいこと
- 並び替え操作が直感的であること

### 5.3 保守性

- 機能ごとに分割された構成であること
- AI と人間の双方が理解しやすいコードであること
- 要件・仕様・コードの対応関係が把握しやすいこと

### 5.4 セキュリティ

- 未認証ユーザーが保護対象データへアクセスできないこと
- ユーザーは参加中プロジェクトのデータだけを扱えること
- 秘密情報をコードに直接埋め込まないこと

### 5.5 拡張性

- 将来的に機能追加しやすいこと
- 話者数の増加、台本構造の拡張に耐えられること

---

## 6. 想定利用シナリオ

### 6.1 基本シナリオ

1. ユーザーがアカウントを作成する
2. ユーザーがログインする
3. プロジェクト一覧を確認する
4. プロジェクトを新規作成する、または既存プロジェクトを開く
5. プロジェクト詳細画面で台本を新規作成する
6. タイトルを設定する
7. 話者を追加し、話者ごとに色を設定する
8. 会話作成画面へ進む
9. 左側フォームから話者とセリフを入力する
10. 必要に応じて画像や URL メディアを追加する
11. 右側で縦線表示された台本とメディアを確認する
12. 必要に応じて任意位置へ挿入する
13. 必要に応じてドラッグアンドドロップで順序を調整する
14. 収録時に台本を見ながら進行する

## 7. データ管理の考え方

### 7.1 データの主体

- 台本データはプロジェクト単位で管理する
- ユーザーは参加しているプロジェクトのデータを扱う

### 7.2 基本データ

- ユーザー
- プロジェクト
- プロジェクト参加者
- 台本
- 話者
- セリフ行
- メディア行
